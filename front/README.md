# 🔐 VPN Server Management System

Полнофункциональное веб-приложение для управления VPN-серверами с учётом оплаты, авторизацией и настройками email-уведомлений.

## 📦 Технологический стек

- Современное SPA‑приложение на **React + TypeScript + Tailwind CSS**
- Backend на **Go + SQLite** с REST API

---

## 🚀 Возможности

### ✅ Авторизация
- Вход в систему с учетными данными
- **Логин**: `admin`
- **Пароль**: `admin123`
- Защищенный доступ ко всем функциям

### 📊 Управление серверами
- ➕ **Добавление** новых серверов
- ✏️ **Редактирование** существующих серверов
- 🗑️ **Удаление** серверов
- 🔗 **Ссылки на хостинги** для каждого сервера (DigitalOcean, Linode, Vultr и др.)

### 💰 Финансовый учёт
- Автоматический расчёт **годовой стоимости**
- Поддержка разных периодов оплаты:
  - Ежемесячно
  - Ежеквартально
  - Ежегодно
- Сводная информация о **суммарных расходах**
- Валюта: **Рубли (₽)**

### 🔔 Уведомления о платежах
- Визуальные подсказки о предстоящих платежах
- Цветовая индикация срочности:
  - 🔴 **Просрочено** - платёж просрочен
  - 🟠 **Срочно** - осталось ≤3 дней
  - 🟡 **Скоро** - осталось 4-7 дней
- Список платежей на ближайшие 30 дней

### 📧 SMTP настройки
- Настройка email-уведомлений
- Конфигурация SMTP-сервера:
  - Хост и порт
  - Учетные данные
  - Адреса отправителя и получателя
- Тестовая отправка писем

### 🔍 Фильтрация и поиск
- **Поиск** по названию, провайдеру, локации или IP
- **Фильтры** по статусу:
  - Все
  - Активные
  - Неактивные
  - Ожидание
- **Сортировка** по:
  - Названию
  - Локации
  - Стоимости
  - Дате платежа

### 📈 Визуализация данных
- Столбчатая диаграмма годовых расходов
- Карточки со статистикой
- Интерактивные элементы

## 🛠️ Frontend (React)

### Технологии
- **React 18** - UI библиотека
- **TypeScript** - типизация
- **Tailwind CSS v4** - стилизация
- **Recharts** - графики
- **Lucide React** - иконки
- **Vite** - сборщик

### Локальный запуск только фронтенда (dev)

```bash
cd front
npm install
npm run dev
```

Фронтенд ожидает доступный backend по пути `/api/...` (в dev‑режиме его можно запускать отдельно).

## 📧 Email-уведомления

Настройки SMTP сохраняются локально. Для реальной отправки email требуется:

1. **Backend сервер** с поддержкой SMTP
2. Библиотека отправки email (например, Nodemailer)
3. Валидный SMTP сервер (Gmail, SendGrid и т.д.)

### Пример настройки для Gmail

```
Хост: smtp.gmail.com
Порт: 587
Имя пользователя: your-email@gmail.com
Пароль: app-specific-password
```

⚠️ Для Gmail требуется [пароль приложения](https://support.google.com/accounts/answer/185833)

## 📝 Использование

### Добавление сервера

1. Нажмите **"Добавить сервер"**
2. Заполните обязательные поля (*)
3. Опционально добавьте ссылку на хостинг
4. Нажмите **"Добавить сервер"**

### Редактирование

1. Нажмите иконку **✏️** на карточке
2. Измените данные
3. Нажмите **"Сохранить изменения"**

### Настройка уведомлений

1. Нажмите **"Настройки"** вверху справа
2. Включите email-уведомления
3. Введите параметры SMTP
4. Нажмите **"Сохранить настройки"**
