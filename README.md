# Биллинг серверов (Server Billing) by Annonyx

Веб‑приложение для учёта серверов, их стоимости, предстоящих платежей и отчётности. Frontend на React, backend на Go, БД SQLite.

## Возможности

- **Управление серверами**: добавление, редактирование, удаление.
- **Финансовый учёт**: месячная и годовая стоимость, периоды оплаты (месяц / квартал / год).
- **Подтверждение оплаты**: отметка «оплачено» с датой и суммой; дата следующего платежа пересчитывается автоматически.
- **Напоминания по email**: настраиваемое количество дней до срока (по умолчанию 10); письмо отправляется автоматически с бэкенда.
- **Отчётность**: расходы по серверам за период, выгрузка в CSV.
- **Авторизация**: вход по логину и паролю; учётные данные задаются только переменными окружения (в коде нет дефолтов).

## Как работает авторизация

1. Логин и пароль **не хранятся** в фронтенде — только ввод и отправка запроса `POST /api/login`.
2. Бэкенд сверяет введённые данные с переменными окружения `ADMIN_USERNAME` и `ADMIN_PASSWORD`. Если они не заданы, вход возвращает ошибку (сервис не настроен).
3. При успешном входе бэкенд возвращает **токен**. Фронтенд сохраняет его локально и передаёт в заголовке `Authorization: Bearer <токен>` при каждом запросе к API.
4. Токен задаётся переменной `AUTH_TOKEN` (или значением по умолчанию в коде для простых установок). Все защищённые эндпоинты проверяют этот заголовок; без токена или с неверным возвращается 401.

## Стек

- **Backend**: Go, SQLite, `net/http`, отправка писем через SMTP.
- **Frontend**: React 18, TypeScript, Tailwind CSS v4, Recharts, Vite.
- **Развёртывание**: Docker, docker compose (один сервис).

## Структура проекта

- `backend/cmd/server` — точка входа.
- `backend/internal/models` — модели (Server, Payment, SMTPSettings, ReportRow).
- `backend/internal/storage` — работа с SQLite (сервера, платежи, SMTP, напоминания).
- `backend/internal/mailer` — отправка писем (напоминания, тест).
- `backend/internal/server` — HTTP‑обработчики и маршруты.
- `front/` — React‑приложение.

## Быстрый старт (Docker)

В `docker-compose.yml` обязательно задайте `ADMIN_USERNAME` и `ADMIN_PASSWORD` (и при необходимости `AUTH_TOKEN`).

```bash
docker compose build
docker compose up -d
```

Приложение: `http://localhost:8080`.

## Локальный запуск

Backend (обязательно задайте переменные окружения):

```bash
cd backend
export ADMIN_USERNAME=myuser
export ADMIN_PASSWORD=mypass
go test ./...
go run ./cmd/server
```

Frontend:

```bash
cd front
npm install
npm run dev
```

Подробное развёртывание с Nginx и доменом — в [Deploy.md](Deploy.md).
